# TryHackMe Scenarios Write-ups

## Overview
This directory contains write-ups for scenario-based exercises completed on the TryHackMe platform.
These scenarios simulate real-world security incidents and are designed to develop practical skills in threat detection, investigation, and response from a SOC Analyst perspective.

## Purpose
- Demonstrate incident analysis and investigation skills
- Apply a structured, SOC-oriented methodology
- Develop hands-on experience with realistic attack scenarios
- Showcase the ability to identify, analyze, and respond to security events

## Structure
Each scenario is organized into its own dedicated folder.

Each write-up typically includes:

- Scenario context and objectives
- Investigation methodology
- Data analysis (logs, network traffic, system artifacts)
- Identification of suspicious or malicious activity
- Findings and conclusions
- Lessons learned

## Skills Demonstrated
Through these scenarios, I am developing:

- Incident investigation and triage
- Log analysis and correlation
- Network traffic analysis
- Threat detection and basic threat hunting
- Understanding of attacker behavior and techniques
- Use of SOC tools (e.g., SIEM, Wireshark, Linux utilities)

## Approach
Each scenario is approached using a structured methodology inspired by real SOC workflows:

1. Initial triage and understanding of the context  
2. Identification of relevant data sources  
3. Analysis and correlation of events  
4. Detection of anomalies or indicators of compromise (IOCs)  
5. Conclusion and reporting  

## Notes
- These write-ups reflect my personal analysis and reasoning process
- Alternative investigation paths may exist
- Content is continuously updated as I improve my skills

## Disclaimer
These write-ups are intended for educational purposes only and may contain spoilers for TryHackMe scenarios.
