# LetsDefend Alert Investigations


## Overview
This directory contains write-ups of alert investigations completed on the LetsDefend platform, specifically from the **Practice** section.
Each investigation simulates a real-world Security Operations Center (SOC) workflow, where alerts must be triaged, analyzed, and properly classified to determine whether they represent legitimate threats or false positives.
The goal of this section is to replicate the daily work of a SOC Analyst, focusing on incident detection, log analysis, and decision-making under uncertainty.


## Purpose
- Simulate real SOC alert triage and investigation processes
- Develop practical skills in threat detection and incident response
- Improve the ability to analyze logs and correlate security events
- Strengthen decision-making in classifying security incidents
- Build a structured investigation methodology


## Investigation Workflow
Each alert investigation typically follows this process:

- Alert overview and context understanding
- Initial triage and severity assessment
- Log and event analysis
- Timeline reconstruction (when applicable)
- Identification of Indicators of Compromise (IOCs)
- Correlation of security events
- Root cause analysis
- Final verdict (True Positive / False Positive / Benign)
- Response recommendations (if applicable)
- Key takeaways


## Skills Demonstrated
Through these investigations, I am actively developing:

- SOC alert triage and prioritization

- Threat detection and validation
- Incident investigation and reasoning
- IOC extraction and analysis
