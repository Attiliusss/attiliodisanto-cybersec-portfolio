# LetsDefend Challenge - Malicious Doc Analysis

## Overview

This write-up documents the analysis of a malicious document challenge from **LetsDefend**.  
The investigation was performed using **VirusTotal** to identify the exploit used, the malware downloaded, the network indicators, and the files dropped on the victim machine.

---

## Tool Used

- **VirusTotal**

---

# Analysis

## Questions 1 & 2

### Q1: What type of exploit is running as a result of the relevant file running on the victim machine?

**Answer:** `rtf.exploit`


### Q2: What is the relevant Exploit CVE code obtained as a result of the analysis?

**Answer:** `CVE-2017-11882`


### Analysis

By analyzing the file information on VirusTotal, the main page provides details about the exploit detected.  
The document is identified as an RTF exploit and is associated with the vulnerability **CVE-2017-11882**, a known Microsoft Office vulnerability related to the Equation Editor component.

![CVE Information](cve.png)

---

# Question 3

### Q: What is the name of the malicious software downloaded from the internet as a result of the file running?

**Answer:** `jan2.exe`


### Analysis

In the **Relations** section of VirusTotal, it is possible to identify the external resources contacted by the malicious document.  
Among the URLs accessed during execution, the downloaded malicious file name is visible: `jan2.exe`


![Downloaded Malware](download_malware.png)

---

# Questions 4 & 5

## Q4: What is the IP address and port information it communicates with?

**Answer:** `185.36.74.48:80`


### Analysis

The **Behavior** section of VirusTotal provides information about the network activity generated by the sample.  
The IP traffic section shows the communication between the malware and the remote server: `185.36.74.48:80`


![TCP Connections](tcp_connections.png)

---

## Q5: What is the exe name it drops to disk after it runs?

**Answer:** `aro.exe`


### Analysis

Inside the **Behavior** section, the **Files dropped** information reveals the executable created on the victim machine after execution: `aro.exe`


![Files Dropped](files_dropped.png)

---

# Takeaways

- The malicious document exploits **CVE-2017-11882**, a vulnerability affecting Microsoft Office documents.
- The file execution triggers the download of an additional malicious payload:
  - `jan2.exe`
- The malware communicates with the following command-and-control infrastructure:
  - `185.36.74.48:80`
- During execution, it drops another executable on the victim machine:
  - `aro.exe`
- VirusTotal's **Relations** and **Behavior** sections are valuable sources for identifying:
  - Downloaded payloads
  - Network indicators
  - Dropped files
  - Malware behavior

---

## Conclusion

This challenge demonstrates how a seemingly harmless document can be used as an initial infection vector.  
By analyzing static and behavioral information through VirusTotal, it is possible to reconstruct the attack chain and extract important Indicators of Compromise (IoCs).
